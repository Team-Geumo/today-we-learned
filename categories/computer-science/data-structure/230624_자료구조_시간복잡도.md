## 시간복잡도
```
시간 복잡도는 알고리즘의 수행 시간이 입력 크기에 대해 어떻게 증가하는지를 나타내는 개념입니다. 알고리즘의 효율성을 평가하는 데 사용되며, 입력 크기가 증가함에 따라 알고리즘의 실행 시간이 어떻게 증가하는지를 분석합니다.

일반적으로, 알고리즘의 시간 복잡도는 입력 크기를 n으로 표기하고, 알고리즘의 실행 시간을 f(n)으로 표기합니다. 여기서 f(n)은 n에 대한 함수입니다.

시간 복잡도는 대개 "최선, 평균, 최악" 세 가지 경우에 대해 나타낼 수 있습니다. 최선의 경우는 입력이 가장 이상적인 경우로, 알고리즘이 가장 효율적으로 실행되는 경우입니다. 평균의 경우는 모든 가능한 입력에 대해 평균적으로 알고리즘이 소요하는 시간을 나타냅니다. 최악의 경우는 입력이 가장 최악인 경우로, 알고리즘이 가장 비효율적으로 실행되는 경우입니다.

시간 복잡도는 일반적으로 "빅오(Big-O) 표기법"을 사용하여 표현됩니다. 빅오 표기법은 알고리즘의 실행 시간을 입력 크기에 대한 상한으로 나타냅니다. 즉, 입력 크기 n이 충분히 크다면 알고리즘의 실행 시간은 O(g(n)) 이하로 유지됩니다. 여기서 g(n)은 어떤 함수이며, n이 증가함에 따라 알고리즘의 실행 시간이 g(n)보다 빠르게 증가하지 않음을 의미합니다.

시간 복잡도를 분석할 때, 주로 빅오 표기법에서 가장 큰 영향을 주는 항을 고려합니다. 일반적으로 상수항은 무시되며, 가장 높은 차수의 항만 고려하여 시간 복잡도를 표기합니다. 예를 들어, O(n^2), O(n log n), O(n), O(log n) 등의 표기가 있습니다.

시간 복잡도 분석을 통해 알고리즘의 효율성을 평가할 수 있으며, 입력 크기에 따른 알고리즘의 실행 시간을 예측하고 비교할 수 있습니다.
```
```
상수 시간 복잡도 (O(1)):
이 경우 알고리즘의 실행 시간이 입력 크기와 무관하게 일정합니다. 예를 들어, 배열에서 첫 번째 요소를 반환하는 알고리즘은 입력 크기에 상관없이 항상 상수 시간이 걸립니다. 배열의 크기가 1개이든 1,000개이든 첫 번째 요소를 찾는 데에는 동일한 시간이 소요됩니다.

선형 시간 복잡도 (O(n)):
이 경우 알고리즘의 실행 시간이 입력 크기에 비례합니다. 예를 들어, 배열에서 특정 값을 찾는 선형 탐색 알고리즘은 배열의 크기에 비례하여 실행 시간이 증가합니다. 배열의 크기가 두 배로 증가하면 탐색 시간도 두 배로 증가합니다.

로그 시간 복잡도 (O(log n)):
이 경우 알고리즘의 실행 시간이 입력 크기의 로그에 비례합니다. 이런 시간 복잡도를 가지는 알고리즘은 입력 크기가 커져도 실행 시간이 상대적으로 느리게 증가합니다. 이진 검색 알고리즘은 이러한 예입니다. 정렬된 배열에서 특정 값을 찾는데 이진 검색을 사용하면 입력 크기가 2배로 증가할 때마다 실행 시간은 한 단계씩 증가합니다.

이차 시간 복잡도 (O(n^2)):
이 경우 알고리즘의 실행 시간이 입력 크기의 제곱에 비례합니다. 이차 시간 복잡도를 가지는 알고리즘은 중첩된 반복문이나 이중 루프를 사용하는 경우입니다. 예를 들어, 배열의 모든 요소 쌍을 확인하는 이중 루프는 이차 시간 복잡도를 가집니다. 배열의 크기가 두 배로 증가하면 실행 시간은 네 배로 증가합니다.
```
